# 地产目标成本系统｜开发总账本

## 一、用途

本文件用于记录地产目标成本系统 V1 开发过程中，每个任务做了什么、由哪个工具 / Agent 执行、当前进度、相关 commit、附件位置和下一步安排。

后续所有 ChatGPT、Codex、Claude、WorkBuddy 参与的任务，都必须在这里形成索引记录。

## 二、工具分工

| 工具 / Agent | 角色 | 主要职责 | 禁止事项 |
| --- | --- | --- | --- |
| 用户 | 最终决策人 | 提需求、看效果、拍板是否继续 | 不需要直接管理所有执行细节 |
| ChatGPT | 总控 / 产品经理 / 验收官 | 主线分流、任务书、产品逻辑、成本逻辑、验收结论、下一步判断 | 不直接改代码 |
| Codex | 程序员 | 按任务书修改代码、运行测试、提交 commit、返回开发回执 | 不自行决定产品逻辑、不超出任务书范围 |
| Claude | 代码审查员 / 第二意见 | 审查复杂逻辑、接口一致性、数据库风险、V1 边界 | 第一阶段不直接提交代码 |
| WorkBuddy | 本地项目助理 | Git 检查、build / lint 检查、截图、打包、回执整理、附件归档 | 不改代码、不改数据库、不提交 Git、不自行判断验收通过 |

## 五、后续记录规则

1. 每个开发任务必须有任务编号。
2. 每个任务必须记录所属主线。
3. 每个任务必须记录执行工具。
4. 每个任务必须记录开始 commit 和完成 commit。
5. 每个任务如有截图、zip、检查报告，必须记录附件路径。
6. 每个任务必须记录下一步交给谁。
7. WorkBuddy 生成的附件统一放入 artifacts/workbuddy/。
8. 文字日志统一放入 docs/dev-log/。
9. 最终验收结论由 ChatGPT 和用户确认。
10. 不允许任何 Agent 自行认定 V1 商用通过。
